# Save a Modified Node

After modifying a node's properties in the path workspace, save your changes using the Save button or **Ctrl+S**.

## What Happens When You Save

- The node's **RecordState** changes from Modified to Unchanged
- The path workspace is updated to reflect the new values
- Other users viewing the same path will see the changes immediately
- The modification timestamp is updated in the system

## Common Issues

**"The end node is not a terminal node"** — This warning appears when the path endpoint is connected to a node that is not configured as a terminal. Check the node type before saving.

**"Unable to save due to validation errors"** — Ensure all required node properties are filled in and valid before saving.

## Verification Steps

After saving the node, verify:
1. The path workspace shows the updated values correctly
2. The node state displays as "Unchanged"
3. Other team members can see the updated node properties

## Next Steps

Once the node is saved, you can:
- Add connectors between nodes to complete the path
- Save the entire path workspace
- Create additional nodes if needed
