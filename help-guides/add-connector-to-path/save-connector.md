# Save a Newly Created Connector

After creating a connector between two nodes, you must save it to persist the changes to your path.

## Saving the Connector

1. **Click the Save button** in the toolbar or press **Ctrl+S**
   - The connector will be validated
   - Any validation errors will be displayed

2. **Review the confirmation**
   - A success message indicates the connector was saved
   - The connector should now appear persistently in the path

3. **Verify the path structure**
   - The connection between nodes is now established
   - The path flow has been updated to include this new connector

## Connector Properties

Before saving, you can configure:
- **Label** — A meaningful name for this transition
- **Type** — The kind of connector (default, conditional, etc.)
- **Validation rules** — Any conditions that must be met

## Common Errors When Saving

**"Terminal node cannot have outgoing connectors"** — The start node is configured as a terminal. Only non-terminal nodes can have outgoing connectors.

**"Connector creates invalid loop"** — The connector would create a problematic cycle. Review the path structure.

**"Duplicate connector detected"** — A connector already exists between these two nodes. Delete the existing one first if you want to replace it.

## After Saving

Once the connector is saved:
1. The path's flow structure is updated
2. Other users can see the connector
3. You can now:
   - Add more connectors between other nodes
   - Save the entire path workspace
   - Run workflow validation to check for errors

## Tips for Creating Good Connectors

- **Clear naming** — Use descriptive labels that explain the transition
- **Logical flow** — Ensure the path makes sense from a process perspective
- **Validation** — Run path validation after adding connectors
